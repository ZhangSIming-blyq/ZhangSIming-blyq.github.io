浅谈 containerd

2022年4月10日
| containerd docker

伴随着kubernetes对docker的弃用，containerd开始进入大众视野；相比于kubelet中集成docker-shim连接docker，docker再次条用containerd去管理容器，直接使用containerd可以通过原生CRI接口的调用实现容器runtime，简化了调用链路，更加的灵活可靠。

一、安装使用 ctr 管理 containerd

# Install Dependent Libraries
$ sudo apt-get update
$ sudo apt-get install libseccomp2

# 下载
# 目前是下载的1.5.2
$ wget https://github.com/containerd/containerd/releases/download/v${VERSION}/cri-containerd-cni-${VERSION}-linux-amd64.tar.gz

# 安装
$ sudo tar --no-overwrite-dir -C / -xzf cri-containerd-cni-${VERSION}-linux-amd64.tar.gz
# 初始化containerd配置
$ containerd config default > /etc/containerd/config.toml
# 修改默认的sandbox_image
$ vim /etc/containerd/config.toml
...
sandbox_image = "registry.cn-beijing.aliyuncs.com/shannonai-k8s/pause:3.1"
...

# 启动服务
sudo systemctl daemon-reload
sudo systemctl start containerd

# 查看版本
$ ctr version

  Version:  1.4.3
  Revision: 269548fa27e0089a8b8278fc4fc781d7f65a939b
  Go version: go1.13.15

二、containerd 使用

2.1 运行一个 busybox 镜像：

  • demo:
# 拉取镜像
$ ctr -n k8s.io i pull docker.io/library/busybox:latest
# 创建一个container(此时还未运行)
$ ctr -n k8s.io container create docker.io/library/busybox:latest busybox
# 创建一个task
$ ctr -n k8s.io task start -d busybox

# 上述步骤也可以简写成如下
$ ctr -n k8s.io run -d docker.io/library/busybox:latest busybox

查看容器在宿主机的 pid，及状态:

$ ctr -n k8s.io task ls
TASK       PID     STATUS
busybox    2356    RUNNING

进入容器：

$ ctr -n k8s.io t exec --exec-id $RANDOM -t busybox sh

杀死移除容器：

$ ctr -n k8s.io t kill -s SIGKILL busybox
$ ctr -n k8s.io t rm busybox
WARN[0000] task busybox exit with non-zero exit code 137

2.2 其他 ctr 命令

镜像标记:

$ ctr -n k8s.io i tag A B
# 若新镜像reference 已存在, 需要先删除新reference, 或者如下方式强制替换
$ ctr -n k8s.io i tag --force A B

删除镜像:

$ ctr -n k8s.io i rm A

拉取镜像:

$ ctr -n k8s.io i pull -k A

查看镜像:

$ ctr -n k8s.io i ls

推送镜像:

$ ctr -n k8s.io i push -k A

导出镜像:

$ ctr -n k8s.io i export A.tar A

导入镜像:

$ ctr -n k8s.io i import A.tar

读取日志: 导出到文件中进行读取:

$ ctr -n k8s.io run --log-uri file:///var/log/xx.log
